Claim form filler

One of the most time consuming and least efficient parts of my current role is submitting a claim form to some rail operators. The forms are clunky, the process time consuming, and it's perhaps the most tedious responsibility shared by our team.

What is this?

The Claim form filler is an experiment in automating input from a CSV file to a web form submission. I was asked to explore automating this following strike action which required us to complete hundreds of forms, and I found it incredibly interesting.

I started out looking at automation gems such as Mechanise and Nokogiri since I was learning Ruby at the time, but started to gravitate towards Python since more tools were available. I was able to create a basic Django server which allowed me to accomplish my original task, but on trying to apply this to other areas, I found that this approach was limiting as some forms reject machine input.


After some research, I came across Selenium WebDriver - an automation framework for testing. I found that by using this, I was able to bypass some of the above restrictions on automated form completion.

By combining Selenium with Tesseract OCR, I was able to automate the completion of the Trenitalia claims form and associated captcha, saving hundreds of man hours compared to submitting claims manually.

Similarly, for completion of the Trenitalia regional claim form, I used Ruby to create a command line tool to prepare claims forms.

This was different to the above as these forms were not a web-form, but required completion of a standardised PDF. I utilised a similar method for iterating through CSV files containing the claim information, but instead used .ERB templates in order to complete and then combine files to create the final PDF, and organised these by case number.

This could still be expanded in order to email the finalised claim form to the respective regional claims hub, but I chose not to do this so as not to run the risk of the receiving parties flagging the emails as spam (although I do intend to investigate this further).

Overall, completing the regional forms in this way allowed for ~250 PDFs to be completed every 4 minutes, as opposed to one form being completed manually in the same way.
