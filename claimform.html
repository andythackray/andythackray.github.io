Claim form filler

One of the most time consuming and least efficient parts of my current role is submitting a claim form to some rail operators. The forms are clunky, the process time consuming, and it's perhaps the most tedious responsibility shared by our team.

What is this?

The Claim form filler is an experiment in automating input from a CSV file to a web form submission. I was asked to explore automating this following strike action which required us to complete hundreds of forms, and I found it incredibly interesting.

I started out looking at automation gems such as Mechanise and Nokogiri since I was learning Ruby at the time, but started to gravitate towards Python since more tools were available. I was able to create a basic Django server which allowed me to accomplish my original task, but on trying to apply this to other areas, I found that this approach was limiting as some forms reject machine input.


After some research, I came across Selenium WebDriver - an automation framework for testing. I found that by using this, I was able to bypass some of the above restrictions on automated form completion.

I'm still in the process of exploring this, but find it really interesting and can't wait to see what else it's capable of.
